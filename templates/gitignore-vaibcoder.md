## Почему именно эти файлы / Why these files

| Категория | Что утечёт | Реальный сценарий |
|-----------|-----------|-------------------|
| **Секреты** | API-ключи, пароли БД, SMTP | `git add .` → `.env.production` в публичном репо → бот украл ключ Supabase за 4 минуты |
| **AI-конфиги** | MCP-токены, серверные URL, чат-история | `.mcp.json` с GitHub PAT → полный доступ к репозиториям |
| **IDE** | debug-конфиги, пути на диске, расширения | `.vscode/launch.json` с `--inspect` и путями к prod-серверу |
| **БД/дампы** | Персональные данные пользователей | `dump.sql` с реальными email и паролями → штраф по 152-ФЗ |
| **Docker** | Токены реестров (DockerHub, GHCR) | `.docker/config.json` → пуш вредоносных образов от твоего имени |

## Как использовать / How to use

1. Скопируй блок выше в `.gitignore` **до первого `git add`**
2. Если уже закоммитил лишнее:
   ```bash
   git rm --cached <файл>
   # Для папки:
   git rm -r --cached <папка>/
   git commit -m "Remove tracked config, update .gitignore"
   ```
3. Если в файле были реальные секреты — **ротируй ключи**, удаление из истории недостаточно
